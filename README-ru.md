# dnd-list-library

**Языки:** [English — README.md](README.md) · Русский (этот файл)

Библиотека для вертикальных списков с drag-and-drop, пружинной анимацией и поддержкой мыши / тача.

## Установка

```bash
npm install dnd-list-library
```

## Быстрый старт

1. Подключите стили (класс `.sortable-list__item`). После публикации в npm:

```ts
import "dnd-list-library/style.css";
```

В **этом репозитории** стили подтягиваются при импорте из [`src/index.ts`](src/index.ts); исходник — [`src/lib/sortable-list.css`](src/lib/sortable-list.css).

2. Создайте контейнер с `position: relative` (библиотека выставит `relative`, если у `root` был `static`).

3. Создайте список:

```ts
import "dnd-list-library/style.css";
import { createSortableList } from "dnd-list-library";

const list = createSortableList({
  root: document.getElementById("list-root")!,
  items: [
    { id: "a", element: elA },
    { id: "b", element: elB, height: 80 },
  ],
  itemWidth: 320,
  paddingTop: 50,
  spring: { k: 290, b: 24 },
  horizontalDragBlend: 1.5,
  onReorder(ids) {
    console.log("новый порядок:", ids);
  },
});

list.getOrder();
list.setOrder(["b", "a"]);
list.destroy();
```

## API

### Экспортируемые имена

| Имя                   | Вид     | Описание                                                                |
| --------------------- | ------- | ----------------------------------------------------------------------- |
| `createSortableList`  | функция | Создаёт контроллер, вешает слушатели, запускает цикл анимации.          |
| `SortableListOptions` | `type`  | Объект опций для `createSortableList`.                                  |
| `SortableListItem`    | `type`  | Одна строка: стабильный `id`, DOM-узел `element`, опционально `height`. |
| `SortableListHandle`  | `type`  | Экземпляр: порядок, программная перестановка, `destroy`.                |

### `createSortableList(options)`

|               |                                                                                                                                                                                                                                                                        |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Сигнатура** | `(options: SortableListOptions) => SortableListHandle`                                                                                                                                                                                                                 |
| **Поведение** | Вешает на элементы класс `sortable-list__item`, при необходимости **добавляет** узлы в `root`, выставляет инлайн `width` / `height`, инициализирует пружины, регистрирует глобальные слушатели на `window` (`resize`, мышь, тач, `pointerdown`) и планирует отрисовку. |

### `SortableListOptions`

| Поле                  | Тип                          | По умолчанию                                                       | Описание                                                                                                                                                                                                        |
| --------------------- | ---------------------------- | ------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `root`                | `HTMLElement`                | —                                                                  | **Обязательно.** Контейнер вёрстки. Если у элемента `position: static`, выставляется `relative`. Координаты `translate` строк — в системе отсчёта этого узла (с учётом `scrollLeft` / `scrollTop`).             |
| `items`               | `SortableListItem[]`         | —                                                                  | **Обязательно.** Начальный стек сверху вниз. Порядок в массиве = начальный визуальный порядок.                                                                                                                  |
| `itemWidth`           | `number` (px)                | `offsetWidth` первого элемента, если он положительный, иначе `320` | Фиксированная ширина строки: центрирование, hit-test, `element.style.width`.                                                                                                                                    |
| `paddingTop`          | `number` (px)                | `50`                                                               | Отступ от верха **контентной** области `root` до покоя первой строки (`y.dest`, когда строка не тянется).                                                                                                       |
| `spring`              | `{ k?: number; b?: number }` | по полям: `k = 290`, `b = 24`                                      | Жёсткость `k` и демпфирование `b` пружины для **x**, **y** и **scale** у каждой строки. Можно передать только `k` или только `b` — второе поле возьмёт значение по умолчанию.                                   |
| `horizontalDragBlend` | `number`                     | `1.5`                                                              | При перетаскивании по X позиция задаётся как `x + (центрКолонки - x) / blend`, то есть смещение к центру колонки. **Чем больше** `blend`, тем **слабее** «притяжение» к центру (больше свободы по горизонтали). |
| `onReorder`           | `(ids: string[]) => void`    | —                                                                  | Вызывается при **изменении** порядка id сверху вниз. На старте для начального порядка **не** вызывается. После перестановки пользователем `ids` совпадает с `getOrder()`.                                       |

**Настройка пружины:** значения по умолчанию как в исходном демо; для интуиции см. [react-motion](https://github.com/chenglou/react-motion) и [визуальный подбор параметров](https://chenglou.me/react-motion/demos/demo5-spring-parameters-chooser/).

| Поле `spring` | По умолчанию | Роль                                                      |
| ------------- | ------------ | --------------------------------------------------------- |
| `k`           | `290`        | Жёсткость — выше, быстрее «прилипание» к целевой позиции. |
| `b`           | `24`         | Демпфирование — выше, меньше колебаний / вылетов за цель. |

### `SortableListItem`

| Поле      | Тип           | Обязательно | Описание                                                                                                                                                                                     |
| --------- | ------------- | ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`      | `string`      | да          | Стабильный идентификатор; должен быть **уникален** среди `items`. Используется в `getOrder`, `setOrder`, `onReorder`.                                                                        |
| `element` | `HTMLElement` | да          | Видимая строка. Если узел ещё не внутри `root`, он **добавляется** в `root`.                                                                                                                 |
| `height`  | `number` (px) | нет         | Высота строки. Если не задана, один раз читается **`offsetHeight`** узла на момент создания; последующее изменение высоты в DOM **автоматически не подхватывается** без пересоздания списка. |

### `SortableListHandle`

| Член       | Тип                       | Описание                                                                                                                                                                                                                                                                                                                                                                                                            |
| ---------- | ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `getOrder` | `() => string[]`          | Текущие id **сверху вниз** (порядок отрисовки стека).                                                                                                                                                                                                                                                                                                                                                               |
| `setOrder` | `(ids: string[]) => void` | Переставляет строки в порядке `ids`. **Ничего не делает**, если длина `ids` не совпадает с числом строк, есть неизвестный id или набор id не совпадает с текущим мультимножеством. Обновляет внутреннее состояние и планирует кадр. После **успешного** `setOrder` **`onReorder` не вызывается** (внутренняя «последняя известная» последовательность приводится к новому порядку; реагируй сам при необходимости). |
| `destroy`  | `() => void`              | Снимает слушатели с `window`, останавливает планировщик rAF, сбрасывает `root.style.cursor`. **Не удаляет** DOM-строки — узлы остаются у вызывающего кода.                                                                                                                                                                                                                                                          |

## Ограничения

- Ориентировано на один вертикальный столбец фиксированной ширины элементов.
- Глобальные слушатели `resize`, `mouseup`, `touchend`, `mousemove`, `touchmove`, `pointerdown` вешаются на `window`.
